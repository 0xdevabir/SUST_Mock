# CRM Ticket Sorter

**SUST CSE Carnival 2026 — QueueStorm Warmup**

CRM Ticket Sorter is a FastAPI microservice that automatically triages fintech customer support tickets. Incoming tickets are analyzed with Anthropic Claude and routed by case type, severity, and department so agents spend less time on manual sorting and high-risk cases (such as fraud or wrong transfers) are escalated faster.

## Local Setup

```bash
# 1. Clone the repository
git clone https://github.com/your-username/crm-ticket-sorter.git
cd crm-ticket-sorter

# 2. Create and activate a virtual environment
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Configure environment variables
cp .env.example .env
# Edit .env and set your Anthropic API key:
# ANTHROPIC_API_KEY=your_key_here

# 5. Start the development server
uvicorn main:app --reload --port 8000
```

Interactive API docs: [http://localhost:8000/docs](http://localhost:8000/docs)

## Docker (Mac, Windows, Linux)

Works on any OS with [Docker Desktop](https://www.docker.com/products/docker-desktop/) (or Docker Engine + Compose on Linux). The app always listens on port `8000` inside the container; you map a host port with `HOST_PORT`.

From the **repository root** (the folder that contains `docker-compose.yml`):

**macOS / Linux**

```bash
cp .env.example .env
# Edit .env and set ANTHROPIC_API_KEY

docker compose up --build
```

**Windows (PowerShell)**

```powershell
Copy-Item .env.example .env
# Edit .env and set ANTHROPIC_API_KEY

docker compose up --build
```

**Windows (Command Prompt)**

```cmd
copy .env.example .env
docker compose up --build
```

Verify:

```bash
curl http://localhost:8000/health
```

Change the host port by setting `HOST_PORT` in `.env` (e.g. `HOST_PORT=3000` → `http://localhost:3000`).

Stop the stack: `docker compose down`

## API Reference

### `GET /health`

Health check endpoint.

**Response**

```json
{
  "status": "ok",
  "service": "crm-ticket-sorter"
}
```

---

### `POST /sort-ticket`

Classifies a customer support ticket and returns routing metadata.

**Request** (`TicketRequest`)

```json
{
  "ticket_id": "BK-2026-00421",
  "channel": "app",
  "locale": "bn",
  "message": "আমি ভুল নম্বরে ৫০০০ টাকা পাঠিয়ে ফেলেছি, এখন কী করব?"
}
```

**Response** (`TicketResponse`)

```json
{
  "ticket_id": "BK-2026-00421",
  "case_type": "wrong_transfer",
  "severity": "high",
  "department": "dispute_resolution",
  "agent_summary": "Customer reports sending 5,000 BDT to an incorrect mobile number via the app and is requesting guidance on next steps.",
  "human_review_required": false,
  "confidence": 0.94
}
```

> **Note:** `agent_summary` is generated by Claude and will vary per request. `human_review_required` is computed server-side and is `true` when `severity` is `critical` or `case_type` is `phishing_or_social_engineering`.

#### Supported field values

| Field | Allowed values |
|-------|----------------|
| `channel` | `app`, `sms`, `call_center`, `merchant_portal` |
| `locale` | `bn`, `en`, `mixed` |
| `case_type` | `wrong_transfer`, `payment_failed`, `refund_request`, `phishing_or_social_engineering`, `other` |
| `severity` | `low`, `medium`, `high`, `critical` |
| `department` | `customer_support`, `dispute_resolution`, `payments_ops`, `fraud_risk` |

## Deployment on Render

1. Push this repository to a **public** GitHub repo.
2. Go to [render.com](https://render.com) → **New** → **Web Service** → connect your repo.
3. Configure the service:

   | Setting | Value |
   |---------|-------|
   | **Build command** | `pip install -r requirements.txt` |
   | **Start command** | `uvicorn main:app --host 0.0.0.0 --port $PORT` |

4. Under **Environment**, add:

   | Key | Value |
   |-----|-------|
   | `ANTHROPIC_API_KEY` | Your Anthropic API key |

5. Click **Deploy**. Render will assign a public URL (e.g. `https://crm-ticket-sorter.onrender.com`).

Verify deployment:

```bash
curl https://your-service.onrender.com/health
```

## LLM

Classification is powered by **Claude Sonnet 4.6** (`claude-sonnet-4-6`) through the official [Anthropic Python SDK](https://github.com/anthropics/anthropic-sdk-python). The model returns structured JSON with case type, severity, department, and a neutral agent summary. Sensitive credentials (PIN, OTP, passwords, card numbers) are never echoed in summaries.
